## 🚀 Deployment & Infrastructure

### Vercel Deployment Configuration

**vercel.json (Critical for React Router)**

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/" }
  ]
}
```

> ⚠️ **This rewrite rule is essential.** Without it, direct URL navigation (e.g., opening `/about` directly) returns a 404 error because Vercel would look for a file at that path. The rewrite sends all routes to `index.html`, allowing React Router to handle routing client-side.

**Deployment Commands:**

```bash
# Install Vercel CLI globally
npm install -g vercel

# Login to Vercel
vercel login

# Deploy to preview
vercel

# Deploy to production
vercel --prod
```

**CI/CD Pipeline:**

```
Developer pushes to GitHub (main branch)
            ↓
GitHub triggers Vercel webhook
            ↓
Vercel pulls latest code
            ↓
npm run build (Vite production build)
            ↓
Static files deployed to Vercel Edge Network
            ↓
Global CDN updated (all edge locations)
            ↓
Deployment complete (typically < 60 seconds)
```

**Contact.jsx Implementation:**

```javascript
import emailjs from '@emailjs/browser';

// Configuration constants
const SERVICE_ID  = '';   // From EmailJS Dashboard
const TEMPLATE_ID = ''; // From EmailJS Dashboard
const PUBLIC_KEY  = '';  // From EmailJS Account

const handleSubmit = async (e) => {
  e.preventDefault();
  setLoading(true);

  const templateParams = {
    from_name:  formData.name,
    from_email: formData.email,
    service:    formData.service,
    message:    formData.message,
  };

  try {
    await emailjs.send(SERVICE_ID, TEMPLATE_ID, templateParams, PUBLIC_KEY);
    setStatus('success');
  } catch (error) {
    setStatus('error');
  } finally {
    setLoading(false);
  }
};
```

**EmailJS Dashboard Template Variables:**

```

{{from_email}}  — Sender's email address
{{service}}     — Selected service of interest
{{message}}     — The enquiry message body
```

**EmailJS Setup Steps:**

```

3. Create Email Template with above variables
4. Copy Service ID, Template ID, and Public Key
5. Add origin whitelist (your domain) in Account settings
6. Paste IDs into Contact.jsx constants
7. Test form on localhost — emails deliver immediately
```

<summary><strong>Q03. How does the contact form work technically?</strong></summary>

The contact form uses the **EmailJS browser SDK**. When a visitor submits the form, React captures the field values and calls `emailjs.send()` with three parameters: Service ID, Template ID, and Public Key. EmailJS routes the message through a configured email service (Gmail in this case) and delivers a formatted email to the agency inbox. No backend server or database is required.

</details>

<details>
<summary><strong>Q04. How do job applications work?</strong></summary>

The Careers page links to a **Google Form** configured with fields for name, email, phone, role, experience, CV upload, portfolio URL, and cover letter. When a candidate submits, the data is automatically logged to a **Google Sheet** (one row per submission) and an email notification is sent to the HR team. The entire pipeline is serverless and free.

</details>

<details>
<summary><strong>Q05. Why React + Vite instead of Next.js?</strong></summary>

React + Vite was chosen because: (1) this is a frontend-only showcase without the need for server-side rendering or static site generation, (2) Vite delivers an extremely fast development experience with sub-second hot module replacement, and (3) the Vercel hosting works seamlessly with Vite SPA deployments. Next.js would be considered if SSR/SEO requirements increase significantly.

</details>

<details>
<summary><strong>Q19. What is the vercel.json rewrite rule for?</strong></summary>

The rewrite rule `{ "source": "/(.*)", "destination": "/" }` is essential for React Router to work on Vercel. Without it, navigating directly to a URL like `/about` tells Vercel to look for a file at that path — which doesn't exist, causing a 404. The rewrite rule forwards all requests to `index.html`, where React Router takes over and renders the correct page.

</details>
